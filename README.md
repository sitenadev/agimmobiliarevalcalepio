# agimmobiliarevalcalepio

Jekyll website for Agenzia Immobiliare Valcalepio.
Live at [https://agenziavalcalepio.bg.it/](https://agenziavalcalepio.bg.it/)

## Directory structure

```
agimmobiliarevalcalepio/
  .github/
    workflows/            ← GitHub Actions
  _data/                  ← site data (YAML/JSON)
  _includes/              ← reusable template partials
  _layouts/               ← page layouts
  _sass/                  ← SCSS source files
    vendor/               ← generated by Gulp (do not edit)
      bootstrap/          ← Bootstrap SCSS
      google/             ← Google Fonts SCSS partial
  _scripts/               ← JS source files
  _site/                  ← Jekyll build output (do not commit)
  assets/
    css/                  ← compiled CSS (commit in production)
    fonts/                ← downloaded fonts (commit)
    img/                  ← static images
    js/                   ← compiled JS (commit in production)
  .env                    ← secrets (do not commit)
  .gitignore
  _config.yml
  favicon.ico
  fonts.list
  gulpfile.js
  index.md
  package.json
  README.md
  robots.txt
```

## Requirements

- Ruby + Bundler (`gem install bundler jekyll`)
- Node.js
- `gulp` cloned as a sibling folder

For Gulp setup and global dependency installation see the [gulp README](https://github.com/sitenadev/gulp/blob/master/README.md).

The expected folder layout on your machine:

```
projects/
  gulp/
  agimmobiliarevalcalepio/
```

## Setup

Clone the repository:

```bash
git clone https://github.com/sitenadev/agimmobiliarevalcalepio.git
cd agimmobiliarevalcalepio
```

Install Ruby dependencies:

```bash
bundle install
```

Retrieve the `.env` file (contains secrets, not versioned) and place it in the project root.

## Serve

```bash
npm run serve
```

Open [http://localhost:8080](http://localhost:8080) in your browser.

## Build

```bash
npm run build
```

Generates the static site in `_site/`.

## Asset pipeline

Gulp handles SCSS compilation, JS bundling, CDN downloads, and Google Fonts. All tasks and configuration options are documented in the [gulp README](https://github.com/sitenadev/gulp/blob/master/README.md).

Production assets (minified, `.min` suffix) must be compiled locally and committed before pushing to GitHub Pages:

```bash
npm run build:assets
git add assets/
git commit -m "Update production assets"
git push
```

GitHub Pages sets `JEKYLL_ENV=production` automatically at deploy time.

## Technical notes

### Hosting

The site is hosted on GitHub Pages using the standard (dependency-based) version of Jekyll 3. Local development requires Jekyll installed on your machine.

### GitHub Actions

Two workflows run automatically:

1. Deploy and property scraper (`.github/workflows/scheduled-github-pages-build.yml`)
Triggers on push to `master`, manually, and on a schedule twice a day. Before building the site it runs a Python script that scrapes the latest property listings from the agency website and writes them to `_data/properties-api.json`, which Jekyll uses at build time to render the listings page.

2. Keepalive (`.github/workflows/scheduled-github-actions-keepalive.yml`)
Runs on the 1st of each month. Prevents GitHub from disabling the scheduled workflows due to repository inactivity, using the [PhrozenByte/gh-workflow-immortality](https://github.com/PhrozenByte/gh-workflow-immortality) action.

This workflow requires a fine-grained personal access token stored as `PERSONAL_ACCESS_TOKEN` in the repository secrets. When it expires, generate a new one following the instructions in the action's documentation, then update the secret in the repository settings.
